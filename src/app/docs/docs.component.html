Angular Performance Optimization

This example app suffers from most performance issues and anti-patterns an Angular app can have.

Most common Angular Performance Problems

  • Too many change-detection cycles and re-renderings
  • Memory: leak or bloat
  • Heavy computations
  • Expensive Event Listeners

Most of the time it is a combination of all of them.

Goals of this Workshop

  • Problem - What is the cause
  • Identification - How to identify with (Angular) DevTools
  • Resolution - How to resolve it in the Demo-App
  • Adaption - How can we apply this to the ALI app
